# 프로젝트 리스트 - 소유/공유 구분 UX 기획서

## 현황 분석

### 현재 상태
- 프로젝트 리스트(`ProjectsView.vue`)에서 내가 만든 프로젝트와 초대받은 프로젝트가 동일한 그리드에 혼재
- 유일한 구분 수단: 카드 하단의 role-badge(마스터/관리자/편집자/읽기전용)뿐
- role-badge는 10px 크기로 작고, "마스터 = 내가 만든 것"이라는 인지 연결이 즉각적이지 않음
- 프로젝트가 많아질수록 "내가 만든 것"과 "참여 중인 것"을 구분하기 어려움

### 사용 가능한 데이터
- `Project.ownerId`: 프로젝트 소유자 ID (API에서 이미 제공)
- `auth.user.id`: 현재 로그인 사용자 ID
- `Project.members[].role`: MASTER, ADMIN, WRITER, READONLY
- `ownerId === auth.user.id`로 소유 여부 판별 가능 (프론트엔드 변경만으로 구현 가능)

---

## 기획 내용

### 핵심 방향
프로젝트 그리드를 **2개 섹션**으로 분리하여, 스크롤 없이도 소유/공유 구분이 한눈에 보이도록 한다.

---

### 1. 섹션 분리 레이아웃

```
[헤더]
[초대 알림 배너] (기존 유지)

프로젝트                              [+ 새 프로젝트]

내 프로젝트 (3)
┌─────────────┐ ┌─────────────┐ ┌─────────────┐
│  카드 A      │ │  카드 B      │ │  카드 C      │
└─────────────┘ └─────────────┘ └─────────────┘

공유받은 프로젝트 (2)
┌─────────────┐ ┌─────────────┐
│  카드 D      │ │  카드 E      │
└─────────────┘ └─────────────┘
```

**섹션 헤더 스펙:**
- 텍스트: "내 프로젝트 (N)" / "공유받은 프로젝트 (N)"
- 폰트: `var(--text-sm)`, `font-weight: 600`
- 색상: `var(--text-tertiary)`
- 섹션 간 간격: `32px`
- 첫 번째 섹션 헤더와 프로젝트 제목("프로젝트") 사이: `20px`
- 섹션 헤더와 그리드 사이: `12px`

**분류 기준:**
- 내 프로젝트: `project.ownerId === auth.user.id`
- 공유받은 프로젝트: `project.ownerId !== auth.user.id`

**빈 상태 처리:**
- 내 프로젝트가 0건: 기존 empty state 표시 (아직 프로젝트가 없습니다)
- 공유받은 프로젝트가 0건: 섹션 자체를 숨김 (빈 섹션을 보여줄 필요 없음)
- 둘 다 0건: 기존 empty state 유지

---

### 2. 공유 프로젝트 카드에 소유자 표시

공유받은 프로젝트 카드에만 **소유자 정보**를 추가 표시한다.

**현재 카드 meta 영역:**
```
[마스터]  멤버 3명  2024년 3월 15일
```

**변경 후 (공유 프로젝트만):**
```
[편집자]  소유자명 님  멤버 3명  2024년 3월 15일
```

**소유자명 스펙:**
- `project.members`에서 `role === 'MASTER'`인 멤버의 `user.username`
- 폰트: `var(--text-xs)`, 색상: `var(--text-disabled)`
- role-badge 바로 오른쪽, 멤버 수 왼쪽에 위치
- 구분자: 자연스러운 gap(8px)으로 분리, 별도 구분선 불필요

---

### 3. 카드 좌측 그래프 패널 시각 차이

내 프로젝트와 공유 프로젝트의 **좌측 그래프 썸네일 영역**에 미묘한 시각 차이를 준다.

**내 프로젝트:**
- 기존 유지 (현재 디자인 그대로)

**공유받은 프로젝트:**
- 좌측 그래프 패널 상단 좌측에 작은 공유 아이콘 오버레이
- 아이콘: 사람+화살표 형태의 간단한 SVG (12x12), 반투명
- 색상: `var(--text-disabled)`, opacity: 0.5
- 위치: 그래프 패널 내부 좌상단 padding 6px

이 아이콘은 보조적인 힌트이며, 주된 구분은 섹션 분리로 달성한다.

---

### 4. 정렬 규칙

각 섹션 내부 정렬:
- 기본: `updatedAt` 내림차순 (최근 수정순)
- 추후 정렬 옵션 확장 가능하나, 현 단계에서는 고정

---

### 5. 스켈레톤 로딩

로딩 중에는 섹션 구분 없이 현재와 동일한 스켈레톤 4개를 표시한다. 데이터가 로드된 후 섹션이 분리된다.

---

## 수정 범위

### 프론트엔드만 수정 (백엔드 변경 없음)

| 파일 | 작업 |
|------|------|
| `client/src/views/ProjectsView.vue` | 섹션 분리 레이아웃, computed 추가, 카드 소유자 표시 |
| `client/src/style.css` | 필요 시 공유 아이콘 관련 CSS 변수 추가 |

### 상세

**ProjectsView.vue `<script>` 추가:**
```typescript
const myProjects = computed(() =>
  projectStore.projects.filter(p => p.ownerId === auth.user?.id)
)
const sharedProjects = computed(() =>
  projectStore.projects.filter(p => p.ownerId !== auth.user?.id)
)

function ownerName(project: Project): string {
  const master = project.members.find(m => m.role === 'MASTER')
  return master?.user.username ?? ''
}
```

**ProjectsView.vue `<template>` 변경:**
- 기존 단일 `projects-grid`를 두 개의 섹션 블록으로 분리
- 각 섹션에 헤더(h2) + 그리드 구성
- 공유 프로젝트 카드의 meta 영역에 `ownerName()` 표시 추가
- 공유 프로젝트 카드 그래프 패널에 공유 아이콘 SVG 추가

## 디자인 가이드 요청 사항 (UXUI 디자이너에게)

### 확인 필요 항목

1. **섹션 헤더 스타일**: "내 프로젝트 (3)" 텍스트의 정확한 weight, size, color 확정. 현재 제안은 `--text-sm` + `--text-tertiary`이나, 페이지 제목("프로젝트")과의 시각적 위계가 자연스러운지 검토

2. **공유 아이콘 디자인**: 좌측 그래프 패널에 들어갈 12x12 공유 아이콘의 정확한 SVG path. 제안하는 형태:
   - 옵션 A: 사람 실루엣 + 우측 화살표 (공유 받음)
   - 옵션 B: 두 사람 겹친 형태 (협업)
   - 옵션 C: 아이콘 없이 섹션 분리만으로 충분한지 판단

3. **공유 프로젝트 카드 hover 색상**: 내 프로젝트와 동일한 주황(accent) hover를 유지할지, 공유 프로젝트는 약간 다른 톤(예: 더 차분한 톤)으로 할지

4. **섹션 접기/펼치기**: 프로젝트 수가 많아질 경우 섹션을 접을 수 있는 토글이 필요한지. 현재 기획에서는 미포함이나, 디자이너 판단에 따라 추가 가능

5. **빈 공유 섹션 처리**: 공유받은 프로젝트가 0건일 때 섹션을 완전히 숨기는 것이 맞는지, 아니면 "공유받은 프로젝트가 없습니다" 같은 빈 상태를 보여줄지

6. **소유자명 표시 형식**: "홍길동 님" vs "홍길동" vs "@홍길동" 중 프로젝트 톤에 맞는 형식

### 제약 조건 (반드시 준수)
- 모든 색상은 `style.css`의 CSS 변수 사용 (하드코딩 금지)
- 이모지 사용 금지
- 기존 카드 레이아웃(좌측 그래프 + 우측 콘텐츠)의 기본 구조는 유지
- role-badge 디자인 시스템은 기존 것을 그대로 사용
