# 작업 이력 - 2026-03-27

## 1. Monorepo 구조 전환

프로젝트를 Client/Server 분리된 풀스택 monorepo 구조로 재편

- `client/` : Vue 3 + Pinia + D3 프론트엔드
- `server/` : Express + Prisma + PostgreSQL 백엔드
- API 레이어 (`client/src/api/`) 추가 - http 클라이언트, 인증/프로젝트/그래프 API

---

## 2. 인증 시스템

JWT 기반 인증 구현

- 회원가입 / 로그인 / 로그아웃 / 토큰 갱신 (access + refresh token)
- Prisma `User`, `Session` 모델
- 라우트 가드 (`authenticate` 미들웨어)
- 화면: LoginView, RegisterView, HeroView

---

## 3. 프로젝트 관리 API

서버 사이드 프로젝트 CRUD

- 프로젝트 생성 / 조회 / 수정 / 삭제
- 멤버 초대 및 권한 관리 (`ProjectMember`)
- 읽기 전용(read-only) 접근 모드 지원

---

## 4. 그래프 데이터 영속화

그래프 노드/의존성을 서버 DB에 저장

- `GraphData` 모델 (data JSON, positions JSON)
- 그래프 저장 / 불러오기 / 노드 위치 저장 API
- Pinia store와 API 연동

---

## 5. AES-256-GCM 암호화 서비스

사용자 개인정보 암호화 저장

- `cryptoService` : AES-256-GCM 암호화/복호화, HMAC-SHA256 해시
- `User.email`, `User.username` 암호화 저장 + `emailHash`, `usernameHash`로 조회/유일성 검증
- 마이그레이션 스크립트: `migrateEncryption.ts`, `migrateUserEncryption.ts`

---

## 6. 감사 로그

사용자 행동 이력 DB 기록

- `AuditLog` 모델 (userId, projectId, action, status, ipAddress, userAgent, failReason)
- 로그인 성공/실패, 로그아웃 이벤트 자동 기록
- 인덱스: projectId+createdAt, userId+createdAt, email, action, status
- `auditLogService.ts` 서비스 레이어

## 20. 멤버 역할(Role) 시스템

프로젝트 멤버 권한을 4단계 역할로 세분화

- `ProjectMemberRole` enum 추가: `MASTER` / `ADMIN` / `WRITER` / `READONLY`
- `ProjectMember.role` 필드 추가 (마이그레이션 포함)
- `projectStore.canWrite` / `canAdmin` computed로 UI 권한 분기
  - `READONLY` 멤버는 편집 모드 전환 불가, Sample 버튼 비노출
  - `ADMIN` 이상만 멤버 관리 버튼 노출
- 멤버 관리 모달 내 역할 변경 / 멤버 추가 시 역할 선택 지원
- 자신보다 높은 역할은 변경·제거 불가 (권한 상위 보호)
- `migrate-roles.ts` 기존 멤버 역할 마이그레이션 스크립트
