# 작업 이력 — 2026-03-28

## 1. 방화벽 오픈 속성 이동 (서버 노드 → 의존성)

방화벽 오픈 요청은 서버 자체의 속성이 아닌 두 서비스 간 연결(의존성)의 속성으로 모델을 재설계했다.

- `Server` 타입에서 `hasFirewall`, `firewallUrl` 필드 제거
- `Dependency`, `D3Link` 타입에 `hasFirewall?`, `firewallUrl?` 필드 추가
- `ServerModal.vue` — 방화벽 체크박스 및 URL 입력 필드 제거
- `DependencyModal.vue` — 방화벽 오픈 필요 체크박스 및 오픈요청 URL 입력 필드 추가 (URL은 체크 시에만 표시)
- `ImpactPanel.vue` — 서버 섹션의 방화벽 표시 제거, 의존하는/받는 노드 목록에 `FW` 뱃지 추가 (URL 있으면 클릭 가능한 링크)
- `ProjectView.vue` — `d3Links` 변환 시 `hasFirewall`, `firewallUrl` 포함하도록 수정

## 2. 의존성 수정 기능 추가

기존에는 의존성을 추가/삭제만 할 수 있었으나 수정 기능을 추가했다.

- `graph.ts` — `updateDependency(id, data)` 함수 추가 (source/target 변경 불가, Undo 지원)
- `DependencyModal.vue` — 편집 모드 추가
  - `editingDep` prop으로 기존 의존성 데이터 수신 및 폼 초기값 세팅
  - 편집 모드에서 source/target disabled 처리
  - 타이틀/버튼 "의존성 수정" / "저장"으로 전환
  - `update` emit으로 수정 결과 전달
  - 수정 모드에서 중복 의존성 체크 제외
- `ImpactPanel.vue` — 의존성 목록 항목에 `✎` 편집 버튼 추가 (hover 시 표시, 파란색)
- `ProjectView.vue` — `openEditDepModal`, `onDepModalUpdate` 핸들러 연결

## 3. 의존성 화살표 더블클릭으로 수정

그래프 캔버스에서 의존성 화살표를 더블클릭하면 수정 모달이 열리도록 했다.

- `GraphCanvas.vue` — 의존성 링크를 `<g>`로 감싸고 `@dblclick.stop` 이벤트 추가
- 얇은 화살표도 쉽게 클릭 가능하도록 `stroke-width="12"` 투명 hit-area 라인 추가
- 읽기 모드가 아닐 때만 `cursor: pointer` 표시
- L7 멤버 연결선은 기존 `pointer-events="none"` 처리로 자동 제외
- `ProjectView.vue` — `@link-dbl-click` 이벤트 수신 후 해당 의존성 수정 모달 오픈 (읽기 모드 guard)

## 4. 의존성 라인에 방화벽 자물쇠 아이콘 표시

`hasFirewall: true`인 의존성 화살표 중간에 앰버색 자물쇠 SVG 아이콘을 표시했다.

- `GraphCanvas.vue` — 라인 중간점 `((x1+x2)/2, (y1+y2)/2)`에 SVG 자물쇠 아이콘 렌더링
- 배경 원(`fill="#1e293b"`)으로 어떤 라인 색상에도 선명하게 보임
- 앰버 색상(`#f59e0b`)으로 방화벽 경고 의미 부여
- `pointer-events="none"` 적용으로 더블클릭 편집 동작에 영향 없음
- `ProjectView.vue` — `d3Links` 변환에 `hasFirewall`, `firewallUrl` 누락 버그 수정
