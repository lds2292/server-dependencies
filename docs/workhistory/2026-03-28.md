방화벽 오픈 요청은 서버 자체의 속성이 아닌 두 서비스 간 연결(의존성)의 속성으로 모델을 재설계했다.

- `Server` 타입에서 `hasFirewall`, `firewallUrl` 필드 제거

- `DependencyModal.vue` — 방화벽 오픈 필요 체크박스 및 오픈요청 URL 입력 필드 추가 (URL은 체크 시에만 표시)
- `ImpactPanel.vue` — 서버 섹션의 방화벽 표시 제거, 의존하는/받는 노드 목록에 `FW` 뱃지 추가 (URL 있으면 클릭 가능한 링크)
- `ProjectView.vue` — `d3Links` 변환 시 `hasFirewall`, `firewallUrl` 포함하도록 수정

## 8. Optimistic Locking 기반 동시 편집 충돌 감지 및 해결

여러 사용자가 동시에 그래프를 편집할 때 발생하는 덮어쓰기 문제를 Optimistic Locking으로 해결했다.

- `schema.prisma` — `GraphData`에 `version` 필드 추가 (저장마다 증가)
- `graphService.ts` — 저장 시 버전 불일치 시 409 반환, 3-way merge로 자동 병합 시도
- `graphController.ts` — 409 충돌 응답 처리 추가
- `graphApi.ts` — version 필드 포함하여 저장 요청, 409 응답 시 충돌 데이터 반환
- `graph.ts` (store) — 충돌 감지 후 자동 병합 또는 수동 해결 플로우 처리
- `GraphConflictModal.vue` — 신규 컴포넌트
  - 충돌 항목별 변경 필드를 diff 테이블로 시각화 (내 변경 / 서버 변경 비교)
  - "내 변경 유지" / "서버 변경 적용" 선택 UI
- `ProjectView.vue` — `GraphConflictModal` 연결
- 저장 트리거를 수동저장/자동저장 타이머/unmount로 한정 (debounce 제거)
- `http.ts` — 로그인 실패(401) 시 인터셉터에서 `/auth/login` 경로 제외로 폼 초기화 버그 수정
- keep-alive 제거로 로그아웃 팝업 잔존 버그 수정
- `crypto.randomUUID` non-secure context fallback 추가
- Prisma 마이그레이션 2건 추가 (`add_audit_log_detail`, `add_graph_version`)
